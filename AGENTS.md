# AGENTS.md

Guidance for AI coding agents working in this repository. Human-oriented docs
live in [README.md](README.md).

## Stack

TanStack Start (SSR) · Vite 8 · React 19 · TypeScript (strict) · Tailwind CSS v4 ·
TanStack Query · Vite+ (`vp`) for lint/format/test · pnpm.

## Golden rules

1. **Use the `vp` toolchain, not ESLint/Prettier/Vitest directly.** Lint and
   format config lives in `vite.config.ts` (`lint`/`fmt` keys), not in
   `.eslintrc`/`.prettierrc`. There are none — don't add them.
2. **Never edit `src/routeTree.gen.ts`.** It's generated by the TanStack Router
   plugin and is excluded from lint/format.
3. **Always run `pnpm check` before declaring work done.** It runs format + lint +
   typecheck together. Use `pnpm check:fix` to auto-fix.
4. **Run `pnpm test`** when you change behavior. Tests live in `src/test/` and
   import the test API from `vite-plus/test` (not `vitest`).
5. **Match the existing style.** Oxfmt defaults: double quotes, semicolons,
   trailing commas, 2-space indent. Let `vp format` decide — don't hand-format.

## Commands

```bash
pnpm dev            # dev server on :3000
pnpm build          # production build (client + SSR)
pnpm check          # format + lint + typecheck (run before finishing)
pnpm check:fix      # same, with auto-fix
pnpm typecheck      # same, with type check
pnpm test           # run tests once
```

## Conventions

- **Imports**: two aliases by purpose — `@/*` → `src/*` for application code
  (`import { seo } from "@/lib/seo"`), and `#/*` → `types/*` for shared type
  declarations (`import type { SeoInput } from "#/seo"`). Put runtime code under
  `src/` (`@/`); put reusable `type`/`interface` declarations under `types/` (`#/`).
- **Layout**: `src/components/default-layout.tsx` wraps every route (from
  `__root.tsx`) with the header/nav/footer. Add nav links via its `navLinks` array.
  Pages render only their own content — don't re-add page chrome per route.
- **Routes**: add files under `src/routes/`. Keep route files thin — put
  components in `src/components/` and import them, so route files only export
  `Route` (route files that export other members trigger a code-splitting warning).
- **Server/API routes**: same file convention, add a `server.handlers` object.
  Use the `[.]` escape for literal dots in paths (e.g. `sitemap[.]xml.ts` → `/sitemap.xml`).
- **SEO**: build meta tags with the `seo()` helper from `@/lib/seo`. Resolve
  absolute URLs with `absoluteUrl()` from `@/lib/site`. Add indexable pages to
  `@/lib/sitemap` (and they appear in `/sitemap.xml`).
- **Env vars**: only `VITE_`-prefixed vars reach the client. Document new ones in
  `.env.example`.

## Testing notes

- Test runner is `vp test` (Vite+'s bundled, Vitest-compatible runner). The
  standalone `vitest` package is intentionally not installed.
- Config is the `test` key in `vite.config.ts` (jsdom env, globals, `src/test/setup.ts`).
- `@testing-library/jest-dom` matchers and globals are typed via `tsconfig.json`
  `types`.

## Deployment (Cloudflare Workers)

- Wired via `@cloudflare/vite-plugin` (in `vite.config.ts`) + `wrangler.jsonc`.
  `pnpm build` emits `dist/client` (assets) and `dist/server` (the Worker, with a
  generated `wrangler.json`).
- The Worker entry is `@tanstack/react-start/server-entry` (set as `main` in
  `wrangler.jsonc`) — don't hand-write a worker file.
- `nodejs_compat` is required in `wrangler.jsonc`. After editing bindings there,
  run `pnpm cf-typegen` to refresh the generated `worker-configuration.d.ts`
  (gitignored).
- The `cloudflare()` plugin is gated behind `!process.env.VITEST` in
  `vite.config.ts` because it's incompatible with the jsdom test environment.
  Keep that guard if you touch the plugin list.
- Scripts: `pnpm deploy` (build + `wrangler deploy`), `pnpm cf-preview`
  (build + `wrangler dev`), `pnpm cf-typegen`.

## What not to do

- Don't add `.eslintrc`, `.prettierrc`, or install `eslint`/`prettier`/`vitest`.
- Don't commit `src/routeTree.gen.ts` edits, `.vite-hooks/_/`, or
  `worker-configuration.d.ts` (all auto-generated).
- Don't introduce a different package manager — this repo is pnpm-only (`packageManager` is pinned).
- Don't create unauthenticated network-exposed endpoints without flagging it.
- Don't remove the `VITEST` guard around `cloudflare()` — it keeps `pnpm test` working.
